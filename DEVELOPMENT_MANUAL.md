## 1. 项目概述

### 1.1 软件定位

**小野兔 Rabbit** 是一款面向写作者的**轻量化 AI 辅助 Markdown 桌面编辑器**。不依赖 Word 或其他办公套件，独立运行，专注于提供沉浸式写作体验。

### 1.2 核心设计原则

| 原则 | 描述 |
|------|------|
| **沉浸写作** | 所有高频操作均可通过快捷键完成，双手不离开键盘 |
| **AI 原生** | AI 辅助是编辑器交互的核心组成部分，非附加功能 |
| **大纲驱动** | 多级标题导航，长文档章节跳转零延迟 |
| **本地优先** | 默认连接本地大模型（llama.cpp/Ollama），可选在线模型 |

### 1.3 目标用户

- 使用 Markdown 写作的长文作者（小说、技术文档、学术论文）
- 需要 AI 辅助进行续写、润色、翻译的写作者
- 在本地部署了大模型的用户
- 使用 DeepSeek、OpenAI、Claude 等在线 AI 的写作者

## 2. 技术架构

### 2.1 技术栈

| 组件 | 选型 | 版本 |
|------|------|------|
| 桌面框架 | Electron | v36.x |
| 前端 | 原生 HTML + CSS + JavaScript | — |
| Markdown 编辑器 | CodeMirror 6 | v6.x |
| MD 渲染 | marked | v15.x |
| 代码高亮 | highlight.js | v11.x |
| 构建打包 | esbuild + electron-builder | v0.25+ / v26.x |

### 2.2 安全模型

- 主进程与渲染进程通过 `preload.js` + `contextBridge` 通信
- `nodeIntegration: false`，`contextIsolation: true`
- 渲染进程中禁止直接调用 Node.js API
- 所有文件系统操作通过 IPC 频道代理

## 4. 基础编辑器

### 4.1 CodeMirror 6 配置

- **语言包**：`@codemirror/lang-markdown`，自动语法着色
- **核心插件**：`basicSetup`（包含行号、括号匹配、折叠、自动补全等）
- **自定义扩展**：暗色主题、自定义快捷键、搜索高亮装饰器、换行控制

### 4.2 主题切换

- **快捷键**：Ctrl+Alt+T 在深色/浅色主题之间切换
- **菜单**：视图 → 深色主题 / 浅色主题，当前激活项显示 ✓
- **编辑器**：主题通过 CodeMirror 6 `Compartment` 动态切换，不重建编辑器实例
- **浅色主题**：
  - 编辑器背景白色，行号栏浅灰
  - 标题和语法着色调整为深色系
  - 预览模式代码块浅灰背景
  - 大纲标题使用深色字体
  - 文件浏览器选中项黑色文字

### 4.3 源码增强模式（默认）

在源码模式下，编辑器对 Markdown 语法元素进行视觉着色增强，而非 Typora 风格的所见即所得：

| 语法元素 | 视觉效果 |
|----------|----------|
| H1 标题 | 1.5em 字号、粗体、蓝色 `#569cd6` |
| H2 标题 | 1.3em 字号、粗体、浅蓝 `#4fc1ff` |
| H3 标题 | 1.15em 字号、粗体、青色 `#4ec9b0` |
| H4 标题 | 1.05em 字号、粗体、黄色 `#dcdcaa` |
| H5 标题 | 正常字号、粗体、橙色 `#ce9178` |
| H6 标题 | 正常字号、粗体、灰色 `#666` |
| **粗体** | 粗体字重 (800)、淡黄色 |
| *斜体* | 斜体、淡橙色 |
| 代码块 | Consolas 等宽字体、0.92em |
| 链接 | 蓝色下划线 `#4fc1ff` |
| 引用 | 绿色、斜体 |
| 删除线 | 灰色删除线 |

### 4.3 预览模式

- **切换**：Ctrl+Shift+P 或点击状态栏"源码/预览"
- **渲染引擎**：marked + highlight.js
- **样式**：GitHub 风格 Markdown 渲染
- **标题颜色**：与源码模式和大纲导航栏一致
- **代码块**：highlight.js 暗色主题语法高亮
- **特点**：只读，不可编辑

### 4.4 字体与换行

| 属性 | 值 |
|------|-----|
| 正文字体 | Microsoft YaHei 16px（默认） |
| 等宽字体 | Consolas |
| 行号 | 显示（14px） |
| 自动换行 | 默认开启，状态栏按钮切换 |
| 光标 | 竖线型，闪烁 |
| IME 输入 | 支持中文输入法正常输入 |

### 4.5 编辑操作 API

```javascript
// 核心 API (editor.js)
Editor.init(container)          // 初始化编辑器
Editor.getContent()             // 获取全文
Editor.setContent(text)         // 设置全文
Editor.getSelection()           // 获取选中区域 {text, from, to, fromLine, toLine, coords}
Editor.getCursorPosition()      // 获取光标位置 {line, column}
Editor.replaceSelection(text)   // 替换选中文本，新内容自动高亮
Editor.insertAfterSelection(t)  // 在选中后方插入文本，新内容自动高亮
Editor.selectLine(lineNumber)   // 选中指定行
Editor.getCurrentLineText()     // 获取光标所在行文本
Editor.getView()                // 获取 CodeMirror EditorView 实例
```

## 5. 文件操作

### 5.1 基础操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 新建 | Ctrl+N | 清空编辑区，标题"未命名.md" |
| 打开 | Ctrl+O | 系统文件对话框，过滤常见文本格式 |
| 保存 | Ctrl+S | 已命名文件直接保存，未命名等同另存为 |
| 另存为 | Ctrl+Shift+S | 系统另存为对话框 |
| 关闭 | Ctrl+W | 未保存弹出确认框（保存/不保存/取消） |
| 退出 | 菜单栏 | 同关闭逻辑 |

### 5.2 支持的文件格式

系统对话框默认显示 Markdown 文件，同时支持打开以下文本文件：

- `.md` `.txt` `.html` `.htm` `.json` `.js` `.css` `.xml` `.yaml` `.yml` `.csv` `.log` `.rst` `.tex` `.py` `.java` `.c` `.cpp` `.h` `.sh`
- 拖拽自动检测，二进制文件（.exe/.zip/.png 等）被过滤

### 5.3 文件浏览器点击行为

- **单击 .md 文件** — 加载内容到编辑器，自动切换到预览模式，焦点停留在文件浏览器
- **单击其他文本文件** — 加载内容到编辑器，源码模式，焦点停留
- **双击任意文件/文件夹** — 进入重命名状态

### 5.4 拖拽打开

- 将文件从系统文件管理器拖入窗口自动加载
- 自动过滤二进制文件并提示
- 未保存内容提示保存确认

### 5.5 最近文件

- 文件菜单下显示最近 5 个文件
- 点击直接打开
- 存储于 `userData/recent_files.json`

### 5.6 自动保存

- 默认每 60 秒自动保存（可在设置中调整 0~300 秒）
- 仅对已命名的文件生效
- 静默保存，不弹提示
- 0 表示关闭自动保存

## 6. 左侧边栏

### 6.1 文件浏览器

**功能**：
- 显示当前文件夹的目录树（响应式展开/收起）
- 文件夹在前、文件在后，按字母序排列
- 点击箭头展开/收起文件夹，Shift+单击展开全部子级
- 不同文件类型显示不同图标（📝 .md / 🌐 .html / 📜 .js / 🎨 .css / 🐍 .py 等）
- 标题栏显示当前路径（截断显示）
- 全部折叠按钮（«）、刷新按钮（↻）
- 选中文件保持蓝色高亮，目录刷新后高亮不丢失

**右键菜单**：
| 操作 | 文件夹 | 文件 |
|------|:------:|:----:|
| 新建文件 | ✓ | — |
| 新建文件夹 | ✓ | — |
| 重命名 | ✓ | ✓ |
| 删除 | ✓ | ✓ |
| 复制路径 | ✓ | ✓ |

**导航**：
- `..` 节点回到上级目录
- 文件菜单"打开文件夹"可切换根目录

### 6.2 大纲导航器

**功能**：
- 自动解析当前文档 H1~H6 标题
- 树形缩进显示多级标题层级
- 点击标题跳转到编辑区对应位置（焦点停留在大纲栏，不切换）
- 预览模式下点击标题平滑滚动预览视图
- 当前光标所在标题自动淡蓝色高亮（`.active`）
- 点击的标题亮蓝色高亮（`.selected`），5 秒后自动消失
- 点击节点左侧箭头展开/折叠子级标题
- Shift+单击箭头展开所有后代节点
- 各级标题用不同颜色区分（与编辑器源码、预览模式一致）：

| 级别 | 颜色 | 字体粗细 |
|------|------|----------|
| H1 | `#569cd6` 蓝色 | 700 |
| H2 | `#4fc1ff` 浅蓝 | 600 |
| H3 | `#4ec9b0` 青色 | 600 |
| H4 | `#dcdcaa` 黄色 | 500 |
| H5 | `#ce9178` 橙色 | 500 |
| H6 | `#666` 灰色 | 500 |

**快捷键**：
| 快捷键 | 功能 |
|--------|------|
| Alt+Shift+1~6 | 大纲折叠到指定级别 |
| Alt+Shift+9 | 展开所有大纲 |

## 7. 右侧 AI 辅助面板

### 7.1 面板结构

```
┌─────────────────────────────────┐
│  [+ 新建对话]                    │ ← 对话历史切换
├─────────────────────────────────┤
│                                 │
│  ┌─────────────────────────────┐│
│  │ 用户 12:30                  ││ ← 用户消息气泡
│  │ 请润色以下内容...           ││
│  │  📋 ✏️ 🗑️ 🔄              ││ ← 图标按钮
│  └─────────────────────────────┘│
│  ┌─────────────────────────────┐│
│  │ AI 12:31                    ││ ← AI 回复气泡
│  │ 润色后的内容...             ││
│  │  📋 📥 📌 🗑️              ││ ← 图标按钮
│  └─────────────────────────────┘│
│                                 │
├─────────────────────────────────┤
│  [输入框 (Enter发送)]           │
│  [模型选择 ▼]    [发送]        │
└─────────────────────────────────┘
```

### 7.2 对话功能

**输入方式**：直接在输入框输入自然语言描述，如：
- "续写以下内容（约 800 字）：..."
- "润色以下内容：..."
- "把这段翻译成英文：..."
- "帮我写一篇关于...的文章"

**发送**：Enter 发送，Shift+Enter 换行

**对话历史**：
- 按文件独立存储（以文件路径 MD5 为 Key）
- 关闭/切换文件时自动保存
- 存储位置：`userData/conversations/<hash>.json`

**新建对话**：点击"+"清空历史

### 7.3 消息操作按钮

**用户消息气泡按钮**：

| 图标 | 功能 | 说明 |
|------|------|------|
| 📋 | 复制 | 复制该条消息内容到剪贴板 |
| ✏️ | 编辑 | 气泡变为输入框修改文字，Enter 重新提交，之后的对话被删除 |
| 🗑️ | 删除 | 删除该条消息，之后的历史对话保留 |
| 🔄 | 刷新 | 从该条消息重新提交，之后的对话被舍弃 |

**AI 回复气泡按钮**：

| 图标 | 功能 | 说明 |
|------|------|------|
| 📋 | 复制 | 复制 AI 回复到剪贴板 |
| 📥 | 替换选中 | 用 AI 回复替换编辑区选中文本 |
| 📌 | 插入后方 | 在编辑区选中文本后方插入 AI 回复 |
| 🗑️ | 删除 | 删除该条回复 |

**全局快捷键**（编辑区和 AI 面板通用）：

| 快捷键 | 功能 |
|--------|------|
| Ctrl+Shift+C | 复制最后一条 AI 回复 |
| Ctrl+Shift+T | 替换选中为最后一条 AI 回复 |
| Ctrl+Shift+I | 插入最后一条 AI 回复到选中文本后方 |
| Alt+L | 聚焦 AI 输入框 |

### 7.4 模型选择

- 发送按钮左侧有模型下拉选择框
- 预置模型列表：`deepseek-chat`、`deepseek-reasoner`、`gpt-4o`、`gpt-4o-mini`、`claude-sonnet-4-20250514`、`local-model`
- 自动将设置中的当前模型加入列表并默认选中
- 切换后立即生效

### 7.5 编辑/刷新逻辑详解

**编辑流程**：
1. 点击用户消息气泡底部的 ✏️ 按钮
2. 气泡内容变为可编辑的 textarea
3. 修改内容后按 Enter → 自动重新提交
4. 该条消息之后的所有历史对话被删除（不保留分叉）

**刷新流程**：
1. 点击用户消息底部的 🔄 按钮
2. 该条消息立即重新提交
3. 该条消息之后的所有历史对话被删除
4. AI 重新生成回复

## 8. Ctrl+K 浮动窗口

### 8.1 触发方式

编辑区按 **Ctrl+K**（有/无选中均可）

### 8.2 窗口结构

```
┌─────────────────────────────────────┐
│ [润色 ▼]  字数: [800]               │
│                                     │
│ 额外指令（可选）:                   │
│ [________________________________]  │
│                                     │
│            [取消]  [确定]           │
└─────────────────────────────────────┘
```

### 8.3 交互流程

1. **有选中** — 弹窗直接操作选中文本
2. **无选中** — 自动选中光标所在行并高亮，然后弹窗
3. 选择模式（润色/续写/定制/英译中/中译英）、设置字数、可选额外指令
4. 点击"确定"或 Ctrl+Enter 发送
5. 弹窗关闭，状态栏显示 "AI 正在生成..."
6. AI 返回后自动替换（润色）或插入（续写）到编辑区
7. 新内容自动保持高亮选中状态

### 8.4 记忆功能

- 上次使用的模式和字数自动记住，下次打开弹窗时恢复

### 8.5 字数精确控制

- 润色模式消息格式：`请润色以下内容（目标约 N 字）：\n\n<选中的文本>`
- 续写模式消息格式：`请续写以下内容（约 N 字）：\n\n<选中的文本>`
- 润色 System Prompt 包含扩充/精简指令

## 9. Ctrl+L 引用到 AI

### 9.1 触发方式

- 编辑区按 **Ctrl+L**
- 有选中：引用选中文本到 AI 对话框
- 无选中：引用光标所在行并自动选中（高亮）

### 9.2 引用格式

输入框中生成：`@文件名 行号-行号 `

例如：`@chapter-01.md 42-42 ` 或 `@chapter-01.md 42-58 `

### 9.3 交互流程

1. 按 Ctrl+L → 引用文本写入 AI 输入框 → 输入框短暂蓝色闪烁
2. 编辑区被引用的行保持高亮选中
3. 用户可在 AI 输入框中追加指令
4. Enter 发送 → AI 处理 → 返回结果
5. 用户可选择替换（Ctrl+Shift+T）或插入（Ctrl+Shift+I）

### 9.4 侧边栏联动

- 如果右侧栏处于隐藏状态，Ctrl+L 自动展开
- 焦点自动切换到 AI 输入框

## 14. IPC 通信协议

### 14.1 频道列表

| 频道 | 方向 | 功能 |
|------|------|------|
| `file:read` | 渲染→主 | 读取文件内容 (UTF-8)，返回 `{success, content}` |
| `file:write` | 渲染→主 | 保存文件内容，返回 `{success}` |
| `file:new` | 渲染→主 | 在目录中创建空 .md 文件，返回 `{success, filePath}` |
| `file:list` | 渲染→主 | 列出目录文件树，返回 `{success, entries[]}` |
| `file:create-dir` | 渲染→主 | 创建新文件夹，返回 `{success, path}` |
| `file:delete-entry` | 渲染→主 | 删除文件或文件夹，返回 `{success}` |
| `file:rename-entry` | 渲染→主 | 重命名，返回 `{success, newPath}` |
| `file:read-multiple` | 渲染→主 | 批量读取文件（@引用），返回 `{success, files}` |
| `dialog:open` | 渲染→主 | 系统打开文件对话框，返回 `{success, filePath}` |
| `dialog:open-folder` | 渲染→主 | 系统打开文件夹对话框，返回 `{success, folderPath}` |
| `dialog:save` | 渲染→主 | 系统另存为对话框，返回 `{success, filePath}` |
| `dialog:confirm-close` | 渲染→主 | 未保存确认对话框，返回 0/1/2 |
| `ai:request` | 渲染→主 | 发起 AI 请求（非流式），返回 `{success, content}` |
| `conversations:load` | 渲染→主 | 加载对话历史，返回 `{success, data}` |
| `conversations:save` | 渲染→主 | 保存对话历史，返回 `{success}` |
| `settings:load` | 渲染→主 | 加载设置，返回设置对象 |
| `settings:save` | 渲染→主 | 保存设置，返回 `{success}` |
| `app:get-paths` | 渲染→主 | 获取 userData/documents 路径 |
| `window:set-mode` | 渲染→主 | 设置窗口模式 (1/2/3) |
| `window:get-mode` | 渲染→主 | 获取当前窗口模式 |
| `window:mode-changed` | 主→渲染 | 推送窗口模式变更通知 |
| `recent:get` | 渲染→主 | 获取最近文件列表 |
| `recent:add` | 渲染→主 | 添加最近文件记录 |
| `app:force-save-and-close` | 主→渲染 | 强制保存并关闭 |

### 14.2 AI 请求格式

```json
// 发送
{
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."}
  ],
  "model": "deepseek-chat",
  "baseUrl": "https://api.deepseek.com/v1",
  "apiKey": "sk-xxx",
  "temperature": 0.7,
  "maxTokens": 2048

// 返回
{ "success": true, "content": "AI 生成的文本..." }
```

- 使用 `stream: false`（非流式），一次性返回完整内容
- 自动拼接 `/chat/completions` 路径
- 支持所有 OpenAI 兼容 API

## 16. AI 模型接入指南

### 16.1 支持的 API 类型

- **llama.cpp** — `http://localhost:8080/v1`
- **Ollama** — `http://localhost:11434/v1`
- **DeepSeek** — `https://api.deepseek.com/v1`（模型：`deepseek-chat` / `deepseek-reasoner`）
- **OpenAI** — `https://api.openai.com/v1`（模型：`gpt-4o`、`gpt-4o-mini` 等）
- **其他兼容 API** — 任何符合 OpenAI Chat Completions 格式的 API

### 16.2 配置方式

**快速配置**：
1. 打开设置（Ctrl+,）
2. AI 标签页
3. 填写 API 基础地址 + API Key + 模型名称
4. 保存

**对话时切换模型**：
- 使用 AI 面板底部的模型下拉选择框

### 16.3 自定义提示词

每个模式都有默认 System Prompt：

| 模式 | 默认提示词 |
|------|-----------|
| 续写 | 专业创意写手，自然续写，保持风格一致 |
| 润色 | 专业中文编辑，提升表达质量，修正语法 |
| 定制 | 全能 AI 助手，按用户要求完成任务 |
| 英译中 | 翻译为流畅自然的中文 |
| 中译英 | 翻译为流畅自然的英文 |

可在设置面板自定义任意模式的提示词。

### 16.4 输出长度控制

- `max_tokens` 是硬上限，模型在此范围内自然停点
- Ctrl+K 在 User 消息中加入了字数指令（如"约 800 字"）
- AI 面板通过在输入框中自然语言指定（如"续写 500 字"）
